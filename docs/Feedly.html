    Feedly

    Talk to the Feedly API. All methods will ensure a valid authentication dance has occurred, and perform the dance if necessary. All of the methods that take a callback also return a promise - the callback is therefore optional. WARNING: by default, this class stores state information such as your access token in ~/.feedly by default.

    new Feedly(options)

    Creates an instance of Feedly.
    Source:
    • feedly.js, line 73
    Parameters:
    Name Type Description
    options Object Options for the API
    Name Type Attributes Description
    port int <optional>
    		 TCP port to listen on for callbacks. (default: 0, which means to pick a random port)
    base String <optional>
    		 The root URL of the API. (default: 'http://cloud.feedly.com')
    config_file String <optional>
    		 File in which state information such as the access token and refresh tokens are stored. Tildes are expanded as needed. (default: '~/.feedly')
    html_file String <optional>
    		 File that contains the HTML to give to the web browser after it is redirected to the one-shot web server that we'll be running. (default: '../html/index.html')
    html_text String <optional>
    		 If html_file is null or the file can't be read, use this text instead. (default: 'No HTML found')
    slop int <optional>
    		 If there is less than this amount of time (in milliseconds) between now and the expiration of the access token, refresh the token. (default: 3600000)
    client_id String The API client ID. (REQUIRED)
    client_secret String The API client Secret. (REQUIRED)
